# Syslog2Bot - 安全设备告警推送系统

> 一款轻量级、跨平台的 Syslog 日志接收与告警推送工具，支持云锁、天眼等主流安全设备，告警消息实时推送到钉钉群。

---

## 🎯 产品简介

在日常安全运营工作中，安全设备产生的告警日志往往分散在各个设备上，运维人员需要频繁登录不同设备查看告警信息，效率低下且容易遗漏重要告警。

**Syslog2Bot** 应运而生！它能够：

- 📥 **集中接收** 多台安全设备的 Syslog 日志
- 🔍 **智能解析** 不同格式的日志内容
- 🎯 **精准筛选** 只推送您关心的告警
- 📢 **实时推送** 告警消息到钉钉群

## ✨ 核心功能

### 1. 多设备支持

| 设备类型 | 支持状态 | 预设模板 |
|---------|---------|---------|
| 云锁安全 | ✅ 已支持 | ✅ 内置 |
| 天眼安全 | ✅ 已支持 | ✅ 内置 |
| 椒图安全 | ✅ 已支持 | - |
| 其他设备 | 🔧 可自定义 | - |

### 2. 智能日志解析

支持多种日志格式解析：

- **Syslog + JSON** - 云锁等设备
- **智能分隔符** - 天眼设备（支持多种告警类型）
- **纯 JSON** - 标准 JSON 格式
- **键值对** - key=value 格式
- **正则表达式** - 非结构化日志

### 3. 灵活筛选策略

支持丰富的筛选条件：

| 操作类型 | 支持的操作符 |
|---------|-------------|
| 比较运算 | `==` `!=` `>` `>=` `<` `<=` |
| 字符串匹配 | `包含` `不包含` `开头匹配` `结尾匹配` |
| 列表匹配 | `包含于` `不包含于` |
| 正则匹配 | `正则表达式` |
| 存在检查 | `字段存在` `字段不存在` |

### 4. 自定义告警模板

支持自定义钉钉消息格式，灵活展示告警信息：

```markdown
### 🚨 安全告警

**告警类型**: {{alertType}}
**告警名称**: {{alertName}}

**攻击IP**: {{attackIP}}
**受害IP**: {{victimIP}}

**告警时间**: {{alertTime}}
**威胁等级**: {{severity}}
**攻击结果**: {{attackResult}}
```

## 📦 安装部署

### 系统要求

**Windows:**
- Windows 10/11 (64位)
- WebView2 运行时（Windows 10+ 通常已内置）

**macOS:**
- macOS 10.15 (Catalina) 或更高版本
- 支持 Intel 和 Apple Silicon (M1/M2/M3)

### 安装步骤

#### Windows

1. 下载 `Syslog2Bot.exe`
2. 双击运行即可
3. 首次运行会在用户目录创建数据库：`C:\Users\用户名\.syslog-alert\syslog.db`

#### macOS

1. 下载 `Syslog2Bot.app`
2. 首次运行：右键 → 打开 → 仍要打开
3. 数据库位置：`~/.syslog-alert/syslog.db`

## 🚀 快速开始

### 第一步：添加钉钉机器人

1. 在钉钉群中添加自定义机器人
2. 安全设置选择"加签"，复制 Secret
3. 复制 Webhook 地址

在 Syslog2Bot 中配置：

1. 进入「数据推送」页面
2. 切换到「机器人管理」标签
3. 点击「添加机器人」
4. 填写机器人名称、Webhook 地址和 Secret

### 第二步：配置解析模板

系统内置了云锁和天眼的解析模板，首次使用时自动创建。

**使用预设模板：**

1. 进入「日志解析」页面
2. 点击「添加模板」
3. 在「预设模板」下拉框选择「云锁」或「天眼」
4. 系统自动填充解析配置
5. 保存即可

### 第三步：配置筛选策略

1. 进入「筛选策略」页面
2. 点击「添加策略」
3. 配置筛选条件，例如：
   - 字段：`severity`
   - 操作符：`包含于`
   - 值：`高危,危急`
4. 选择匹配逻辑：满足任一条件 / 满足全部条件
5. 保存

### 第四步：配置告警策略

1. 进入「数据推送」页面
2. 切换到「告警策略」标签
3. 点击「添加策略」
4. 关联筛选策略、机器人、消息模板
5. 保存

### 第五步：启动服务

1. 进入「系统状态」页面
2. 点击右上角的「启动」开关
3. 服务启动后，监听 UDP 5140 端口

### 第六步：配置安全设备

在安全设备上配置 Syslog 转发：

- **目标地址**：运行 Syslog2Bot 的服务器 IP
- **目标端口**：5140（可在设置中修改）
- **协议**：UDP

## 📖 详细配置指南

### 天眼设备配置

天眼设备支持多种告警类型，Syslog2Bot 采用智能分隔符解析方式：

**支持的告警类型：**

| 告警类型 | 说明 |
|---------|------|
| webids_alert | 网页漏洞利用告警 |
| ioc_alert | 威胁情报告警 |
| ips_alert | 入侵防御告警 |
| webshell_alert | Webshell 告警 |

**配置步骤：**

1. 使用「天眼」预设模板创建解析模板
2. 根据实际日志格式，调整子模板字段位置
3. 开启「跳过头部」选项
4. 配置值转换规则（威胁等级、攻击结果）

**字段位置配置：**

```
webids_alert:
- 告警名称: field_3
- 攻击IP: field_6
- 受害IP: field_8
- 告警时间: field_4
- 威胁等级: field_10
- 攻击结果: field_26

ioc_alert:
- 告警名称: field_18
- 攻击IP: field_6
- 受害IP: field_8
- 告警时间: field_10
- 威胁等级: field_12
```

### 云锁设备配置

云锁设备采用 Syslog + JSON 格式：

**配置步骤：**

1. 使用「云锁」预设模板创建解析模板
2. 无需额外配置，预设模板已包含完整的解析规则

**自动值转换：**

| 字段 | 原始值 | 转换后 |
|------|--------|--------|
| result | 0/1 | 失败/成功 |
| dealStatus | 0-6 | 未处理/已处理(自动)/已处理(手动)/误报/不关注/处置失败/处置中 |

### 筛选条件配置技巧

**示例1：筛选高危和危急告警**

- 字段：`severity`
- 操作符：`包含于`
- 值：`高危,危急`

**示例2：筛选特定 IP 的告警**

- 字段：`attackIP`
- 操作符：`开头匹配`
- 值：`192.168.`

**示例3：排除误报**

- 字段：`alertName`
- 操作符：`不包含`
- 值：`误报`

## 🔧 系统设置

### 基础设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 监听端口 | Syslog 服务监听端口 | 5140 |
| 日志保留天数 | 原始日志保留时间 | 30 天 |
| 告警开关 | 全局告警开关 | 开启 |
| 告警间隔 | 告警去重时间窗口 | 60 秒 |

### 告警去重

系统支持告警去重功能，避免短时间内重复推送相同告警：

- **去重依据**：设备ID + 策略ID + 攻击IP + 威胁类型 + 事件描述
- **时间窗口**：可配置，默认 60 秒

## ❓ 常见问题

### Q1: 日志接收后没有推送告警？

**排查步骤：**

1. 检查日志状态：
   - `pending`：告警处理未执行，检查服务是否启动
   - `unmatched`：筛选条件未匹配，检查筛选策略配置
   - `matched`：已匹配，检查告警策略和机器人配置

2. 检查解析是否成功：
   - 查看 `parsed_data` 字段是否有内容
   - 如果为空，检查解析模板配置

### Q2: 天眼日志解析失败？

**常见原因：**

1. 子模板键名不匹配：确保子模板键名与日志中的告警类型一致（如 `webids_alert`）
2. 未开启跳过头部：如果日志包含 Syslog 头部，需要开启「跳过头部」
3. 字段位置错误：检查字段位置配置是否正确

### Q3: 告警时间显示错误？

如果时间显示为 `58158-01-19`，说明时间戳是毫秒级的，需要更新到最新版本。

### Q4: Windows 运行提示缺少 WebView2？

下载并安装 WebView2 运行时：
https://developer.microsoft.com/en-us/microsoft-edge/webview2/
