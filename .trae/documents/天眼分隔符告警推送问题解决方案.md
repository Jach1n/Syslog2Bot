# 天眼分隔符告警推送问题解决方案

## 问题分析

### 问题1：天眼端口存活检测日志显示为"紧急"

**现象：** 天眼定期推送端口存活检测日志（原始内容：`check the port available`），在日志列表中显示为"紧急"

**根本原因：**
1. 这类日志不是告警日志，而是心跳/探测日志
2. 日志中没有 severity 字段，无法通过解析获取威胁等级
3. 前端显示逻辑优先使用 syslog 协议的 severity（从 priority 解析：`priority % 8`）
4. syslog severity=0 表示"紧急"，这是 syslog 协议定义，与天眼威胁等级无关

**解决方案：**
1. 修改日志显示逻辑，区分 syslog severity 和解析后的威胁等级
2. 添加筛选规则过滤非告警类型日志

### 问题2：解析模板测试无法显示预览

**现象：** 在查看模板处使用告警日志测试时，无法显示解析后的实时预览

**可能原因：**
1. 前端调用 TestParseTemplate API 时参数传递问题
2. 解析类型为 `delimiter` 时，fieldMapping 配置格式问题

### 问题3：推送模板显示异常

**现象：** 推送格式显示：`|!: 0: [object Object]:`

**根本原因：**
用户在输出模板中使用了 `{{delimiter}}`, `{{typeField}}`, `{{subTemplates}}` 这些是**解析配置字段**，不是**解析后的数据字段**。

### 问题4：智能分隔符子模板配置问题

**现象：**
1. 用户编辑智能分隔符模板，配置子模板（如 webids）
2. 添加了自定义字段（威胁等级、攻击结果）
3. 再次编辑该子模板时，之前添加的自定义字段丢失了

**根本原因：**
- 使用浅拷贝导致嵌套对象引用问题
- 默认字段只有4个，需要扩展到6个统一字段

**用户需求：**
- 4个天眼告警子模板需要统一6个字段：**告警名称、攻击IP、受害IP、告警时间、威胁等级、攻击结果**
- 同时支持修改默认字段和批量配置功能

### 3. 修复深拷贝问题

**文件：** `frontend/src/views/ParseTemplates.vue`

```javascript
function editSubTemplate(type: string) {
  currentSubTemplateType.value = type
  // 使用深拷贝
  const config = smartDelimiterConfig.value.subTemplates[type]
  currentSubTemplateConfig.value = JSON.parse(JSON.stringify(config))
  // 确保新字段存在
  if (currentSubTemplateConfig.value.severityField === undefined) {
    currentSubTemplateConfig.value.severityField = -1
  }
  if (currentSubTemplateConfig.value.attackResultField === undefined) {
    currentSubTemplateConfig.value.attackResultField = -1
  }
  subTemplateSampleLog.value = ''
  parsedFields.value = []
  subTemplateDialogVisible.value = true
}

function saveSubTemplate() {
  // 使用深拷贝保存
  smartDelimiterConfig.value.subTemplates[currentSubTemplateType.value] = 
    JSON.parse(JSON.stringify(currentSubTemplateConfig.value))
  subTemplateDialogVisible.value = false
  ElMessage.success('子模板保存成功')
}
```

### 4. 添加批量配置功能

**文件：** `frontend/src/views/ParseTemplates.vue`

```javascript
function applyToAllSubTemplates() {
  const types = Object.keys(smartDelimiterConfig.value.subTemplates)
  if (types.length === 0) {
    ElMessage.warning('没有其他子模板')
    return
  }
  
  ElMessageBox.confirm(
    `确定要将当前字段配置应用到所有 ${types.length} 个子模板吗？`,
    '批量应用',
    { type: 'warning' }
  ).then(() => {
    const currentConfig = {
      alertNameField: currentSubTemplateConfig.value.alertNameField,
      attackIPField: currentSubTemplateConfig.value.attackIPField,
      victimIPField: currentSubTemplateConfig.value.victimIPField,
      alertTimeField: currentSubTemplateConfig.value.alertTimeField,
      severityField: currentSubTemplateConfig.value.severityField,
      attackResultField: currentSubTemplateConfig.value.attackResultField
    }
    
    for (const type of types) {
      if (type !== currentSubTemplateType.value) {
        smartDelimiterConfig.value.subTemplates[type] = {
          ...currentConfig,
          customFields: smartDelimiterConfig.value.subTemplates[type].customFields || []
        }
      }
    }
    ElMessage.success('已应用到所有子模板')
  })
}
```

### 6. 修改 Logs.vue 威胁等级显示

```javascript
function getLevelDesc(log: LogItem): { type: string; text: string } {
  if (log.parsedFields) {
    try {
      const fields = JSON.parse(log.parsedFields)
      if (fields.severity) {
        return getLevelTag(fields.severity)
      }
      if (fields.levelDesc) {
        return getLevelTag(fields.levelDesc)
      }
    } catch (e) {
      console.error(e)
    }
  }
  return { type: 'info', text: '信息' }
}

function getLevelTag(level: string): { type: string; text: string } {
  const levelMap: Record<string, { type: string; text: string }> = {
    '危急': { type: 'danger', text: '危急' },
    '高危': { type: 'danger', text: '高危' },
    '中危': { type: 'warning', text: '中危' },
    '低危': { type: 'info', text: '低危' },
    '信息': { type: '', text: '信息' }
  }
  return levelMap[level] || { type: '', text: level }
}
```

### 7. 添加 in/not_in 操作符

**文件：** `syslog_service.go`

```go
case "in":
    values := strings.Split(cond.Value, ",")
    for _, v := range values {
        if strings.TrimSpace(v) == strValue {
            return true
        }
    }
    return false
case "not_in":
    values := strings.Split(cond.Value, ",")
    for _, v := range values {
        if strings.TrimSpace(v) == strValue {
            return false
        }
    }
    return true
```
